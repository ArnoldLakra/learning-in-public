# Day 2: Internet, Servers, and Linux Basics 🌐🐧

## 1. Networking & Application Basics
*   *How the Internet Works:* A client (like my web browser) sends a request across the internet using the server's IP address, and the server processes that request to send back the required data (like a web page)
*   *What is a Server?:* A computer or software system that provides data, resources, or services to other computers (clients) over a network.
*   *Web Server vs. Application Server:* 
    *   Web Server: Handles simple, static content like HTML pages and images.
    *   Application Server: Handles complex, dynamic business logic and processes data behind the scenes.
*   *Standalone vs. Web Applications:* 
    *   Standalone: Apps installed locally on your PC (like VS Code or VLC player).
    *   Web Apps: Apps that run inside a web browser over the internet (like GitHub or YouTube).
    ## 2. Introduction to Linux Architecture
*   *What is Linux?:* An open-source, highly secure operating system widely used to power cloud servers and infrastructure.
*   *Linux vs. Windows:* Linux is completely open-source, lightweight, highly customizable, and relies heavily on the command line, whereas Windows is closed-source and GUI-focused.
*   *The Core Components:*
    *   *Kernel:* The heart of the OS that directly talks to and controls your computer's hardware.
    *   *Bootloader:* The initial program that runs to load the operating system when the computer is turned on.
    *   *Shell:* The command-line interface (CLI) interpreter that takes the commands you type and passes them to the kernel.

## 💡 Key Takeaway for Cloud Security
In Cloud Security, you rarely get a graphic interface. Understanding how the Linux Kernel, Shell, and Servers interact is fundamental because securing the cloud means securing these exact layers from hackers.
